Security

1. Computer Networking & Linux Administration Fundamentals

Kiến thức về Mạng máy tính:

  • Mô hình OSI (Open Systems Interconnection): 7 tầng (Physical, Data Link, Network, Transport, Session, Presentation, Application).
  • Giao thức TCP/IP: Các lớp mạng cơ bản (Application, Transport, Internet, Network Access).
  • LAN/WAN: Mạng nội bộ (LAN) và mạng diện rộng (WAN).
  • Switching & Routing: Cách chuyển mạch và định tuyến hoạt động.
  • IP Addressing: Địa chỉ IPv4, IPv6, subnetting.
  • NAT, DHCP, DNS: Các giao thức phân phối địa chỉ IP và tên miền.
  • Network Devices: Router, Switch, Hub, Firewall.
  • Wireless Networking: Wi-Fi, chuẩn IEEE 802.11.
  • Firewalls, VPNs: An ninh mạng và kết nối từ xa an toàn.

Kiến thức về Quản trị Linux:

  • Linux Shell: Bash scripting, command line interface (CLI).
  • File Systems: ext4, NTFS, FAT32, phân quyền file và thư mục.
  • User & Group Management: Quản lý người dùng và quyền hạn.
  • Process Management: Chạy, dừng, và kiểm tra các tiến trình.
  • Networking Tools: ifconfig, netstat, ping, traceroute.
  • Package Management: apt, yum, dnf.
  • Services & Daemons: Quản lý các dịch vụ chạy nền, như Apache, MySQL.
  • System Monitoring: Kiểm tra hiệu suất hệ thống bằng top, htop, iostat.

2. CyberSecurity Fundamentals

  • CIA Triad: Confidentiality (Bảo mật thông tin), Integrity (Toàn vẹn dữ liệu), Availability (Khả dụng).
  • Types of Attacks: DoS, DDoS, Man-in-the-Middle, SQL Injection, Phishing.
  • Cryptography: Mã hóa dữ liệu bằng các thuật toán như AES, RSA, SHA-256.
  • Vulnerability Management: Phát hiện và khắc phục lỗ hổng bảo mật.
  • Security Policies: Các chính sách bảo mật như BYOD, Acceptable Use Policy (AUP).
  • Incident Response: Quản lý sự cố bảo mật, bao gồm phát hiện và khắc phục.
  • Network Security: Firewall, Intrusion Detection System (IDS), Intrusion Prevention System (IPS).
  • Endpoint Security: Bảo vệ máy tính cá nhân bằng phần mềm chống virus, phần mềm chống malware.

3. Certified Ethical Hacker v10 (CEH v10)

  • Reconnaissance: Thu thập thông tin (Footprinting), các kỹ thuật như Google Hacking, Whois Lookup, DNS Enumeration.
  • Scanning & Enumeration: Quét mạng (Network Scanning), sử dụng Nmap, Wireshark.
  • Vulnerability Analysis: Phân tích và phát hiện các lỗ hổng bảo mật.
  • Exploitation: Sử dụng các công cụ khai thác như Metasploit.
  • Social Engineering: Tấn công bằng cách lừa đảo thông qua con người.
  • Password Attacks: Các phương pháp tấn công mật khẩu như brute-force, dictionary attacks.
  • Malware: Virus, trojans, worms, rootkits.
  • Web App Security: XSS, SQL Injection, Command Injection.
  • Wireless Network Security: WEP, WPA/WPA2, tấn công wireless.

4. Basic Penetration Testing - ECSA Certified Secure Analyst (v10)

  • Penetration Testing Phases: Thu thập thông tin, phân tích lỗ hổng, khai thác, và viết báo cáo.
  • Tools: Burp Suite, OWASP ZAP, Nmap, Wireshark, Metasploit.
  • Web Application Penetration Testing: XSS, SQL Injection, CSRF, IDOR (Insecure Direct Object References).
  • Network Penetration Testing: Tấn công vào mạng nội bộ và thiết bị.
  • Social Engineering: Tấn công thông qua email phishing, các chiến thuật lừa đảo.
  • Post Exploitation: Xóa dấu vết, tăng quyền admin, thiết lập backdoor.

5. Advanced Web Application Penetration Testing

  • OWASP Top 10: Tìm hiểu về các lỗ hổng phổ biến trong ứng dụng web như XSS, SQL Injection, Insecure Deserialization.
  • Advanced SQL Injection: Bypass authentication, union attacks, blind SQL Injection.
  • Cross-Site Scripting (XSS): Reflected, Stored, DOM-based XSS.
  • Cross-Site Request Forgery (CSRF): Cách thực hiện và bảo vệ khỏi CSRF.
  • Security Misconfiguration: Lỗi cấu hình bảo mật và các cách tấn công.
  • File Upload Vulnerabilities: Tấn công thông qua chức năng tải file lên.
  • Session Management Attacks: Tấn công session hijacking và session fixation.
  • Burp Suite Pro: Sử dụng các tính năng nâng cao như Intruder, Repeater, và Scanner.

6. Computer Hacking Forensic Investigator (CHFI v9)

  • Digital Forensics: Quá trình điều tra các vụ việc tội phạm mạng.
  • Forensic Process: Xác định, bảo tồn, phân tích và báo cáo bằng chứng kỹ thuật số.
  • Data Acquisition: Thu thập dữ liệu từ ổ cứng, thiết bị di động và mạng.
  • File Systems Forensics: Kiểm tra các hệ thống file, phân tích các file ẩn, xóa.
  • Memory Forensics: Điều tra dữ liệu trong bộ nhớ RAM.
  • Network Forensics: Phân tích lưu lượng mạng để phát hiện tấn công và vi phạm.
  • Log Analysis: Phân tích các file log của hệ thống và ứng dụng.
  • Anti-forensics: Các kỹ thuật tấn công viên dùng để xóa dấu vết và cách phát hiện.

Phân Tích Mã Độc Căn Bản (Basic Malware Analysis)

Phân tích mã độc căn bản là quá trình hiểu và khám phá cách mã độc hoạt động mà không cần phải có những kỹ năng hay công cụ quá phức tạp.

1. Phân Tích Tĩnh (Static Analysis)

Đây là bước phân tích mã độc mà không cần thực thi mã:

  • File Identification: Kiểm tra thông tin cơ bản của file như loại file, kích thước, chữ ký.
    • Công cụ: file (Linux), PEiD, TrID.
  • Hashing: Tạo mã băm (hash) của mã độc để kiểm tra xem nó đã được nhận diện trước đây chưa.
    • Công cụ: MD5, SHA256, VirusTotal.
  • String Analysis: Phân tích chuỗi ký tự bên trong file để tìm các đoạn mã đáng ngờ hoặc các thông tin như URL, địa chỉ IP, mật khẩu.
    • Công cụ: strings, BinText.
  • PE Header Analysis: Kiểm tra tiêu đề của file thực thi (Portable Executable - PE) để hiểu cấu trúc và các thư viện mà mã độc có thể sử dụng.
    • Công cụ: PEview, CFF Explorer.
  • Disassembly: Dịch mã máy thành mã Assembly để kiểm tra lệnh mà mã độc sử dụng.
    • Công cụ: IDA Pro, Ghidra, OllyDbg.

2. Phân Tích Động (Dynamic Analysis)

Phân tích động là quá trình thực thi mã độc trong một môi trường kiểm soát để quan sát hành vi của nó:

  • Sandbox Analysis: Chạy mã độc trong một môi trường cách ly (sandbox) để tự động ghi lại hành vi.
    • Công cụ: Cuckoo Sandbox, Any.Run, Joe Sandbox.
  • Process Monitoring: Theo dõi các tiến trình mà mã độc tạo ra hoặc tương tác.
    • Công cụ: Process Monitor, Process Explorer.
  • Registry Monitoring: Kiểm tra xem mã độc có sửa đổi hoặc thêm các khóa registry không.
    • Công cụ: Regshot, Autoruns.
  • Network Traffic Analysis: Quan sát các gói tin mạng mà mã độc gửi đi hoặc nhận về.
    • Công cụ: Wireshark, Fiddler, TCPView.
  • File System Monitoring: Kiểm tra sự thay đổi của hệ thống file, xem mã độc có tạo file mới hay xóa file không.
    • Công cụ: Process Monitor, Sysinternals Suite.

Phân Tích Mã Độc Nâng Cao (Advanced Malware Analysis)

Phân tích mã độc nâng cao yêu cầu kỹ năng sâu hơn trong việc dịch ngược mã (reverse engineering), phân tích lưu lượng mạng phức tạp và nghiên cứu sâu hơn về các kỹ thuật né tránh mà mã độc sử dụng.

1. Kỹ Thuật Dịch Ngược (Reverse Engineering)

  • Decompilation: Dịch mã nhị phân (binary code) ngược lại thành mã nguồn.
    • Công cụ: Ghidra, IDA Pro, Radare2.
  • API Call Analysis: Phân tích các lệnh API mà mã độc sử dụng để tương tác với hệ điều hành hoặc mạng.
    • Công cụ: API Monitor, Sysinternals Suite.
  • Debugging: Sử dụng debugger để dừng và quan sát quá trình thực thi mã độc từng bước.
    • Công cụ: x64dbg, OllyDbg.

2. Phân Tích Mã Độc Ẩn Danh (Obfuscated Malware)

Mã độc nâng cao thường sử dụng các kỹ thuật ẩn danh để tránh bị phát hiện:

  • Packing/Unpacking: Một số mã độc được "gói" để làm phức tạp quá trình phân tích. Kỹ năng unpack là cần thiết để khôi phục mã gốc.
    • Công cụ: UPX, PEiD, unpackme.
  • Code Obfuscation: Mã độc có thể được mã hóa hoặc làm rối để làm khó quá trình đọc mã nguồn.
    • Công cụ: De4Dot, Uncompyle6 (đối với Python).
  • Anti-Debugging/Anti-VM Techniques: Mã độc có thể nhận diện môi trường debug hoặc sandbox và thay đổi hành vi.
    • Công cụ: Debug kỹ thuật với x64dbg, kiểm tra dấu hiệu anti-VM với VMDetect.

3. Phân Tích Mã Độc Khai Thác Lỗ Hổng (Exploitation-based Malware)

  • Zero-day Analysis: Phân tích mã độc khai thác các lỗ hổng chưa được công bố (zero-day).
    • Công cụ: Kiểm tra lỗ hổng với Metasploit, phân tích sâu với các kỹ thuật fuzzing.
  • Shellcode Analysis: Một số mã độc sử dụng shellcode để thực hiện các hoạt động tấn công.
    • Công cụ: Scdbg, SCTEST, Radare2.

4. Phân Tích Lưu Lượng Mạng (Network Forensics)

Phân tích mã độc liên quan đến lưu lượng mạng nâng cao:

  • Protocol Analysis: Kiểm tra các giao thức mạng mà mã độc sử dụng, như HTTP, FTP, DNS.
    • Công cụ: Wireshark, NetWitness.
  • Encrypted Traffic: Phân tích lưu lượng mã hóa mà mã độc sử dụng, như HTTPS hoặc VPN.
    • Công cụ: SSLstrip, mitmproxy.
  • Command and Control (C2): Theo dõi và phân tích hoạt động của mã độc với máy chủ điều khiển từ xa (C2).
    • Công cụ: Wireshark, Suricata, Snort.

5. Phân Tích Bộ Nhớ (Memory Forensics)

Phân tích bộ nhớ là cách tiếp cận nâng cao để phân tích mã độc mà không cần file:

  • Memory Dump: Lấy thông tin từ bộ nhớ của một hệ thống đang chạy để phân tích mã độc.
    • Công cụ: Volatility, Rekall.
  • Code Injection Analysis: Kiểm tra xem mã độc có thực hiện kỹ thuật injection để ẩn mình trong bộ nhớ của các tiến trình khác hay không.
    • Công cụ: Volatility, ProcDump.

Sự Khác Biệt Giữa Phân Tích Căn Bản và Nâng Cao:

  • Phân Tích Căn Bản tập trung vào việc sử dụng các công cụ tự động để phát hiện mã độc và hiểu các hành vi cơ bản của nó. Đây là bước đầu tiên để đánh giá nguy cơ mã độc.
  • Phân Tích Nâng Cao yêu cầu kiến thức sâu hơn về lập trình, dịch ngược mã và các kỹ thuật chuyên sâu để hiểu rõ hơn về mục tiêu và kỹ thuật của mã độc, cũng như cách thức mà nó né tránh các công cụ bảo mật.

Certified Cloud Security Professional (CCSP)

CCSP là chứng chỉ cao cấp dành cho các chuyên gia bảo mật đám mây, tập trung vào việc hiểu và áp dụng các biện pháp bảo mật cho môi trường điện toán đám mây.

1. Cloud Concepts, Architecture and Design (Khái niệm, Kiến trúc và Thiết kế đám mây)

  • Cloud Service Models: Hiểu rõ các mô hình dịch vụ đám mây như SaaS (Software as a Service), PaaS (Platform as a Service), IaaS (Infrastructure as a Service).
  • Cloud Deployment Models: Các mô hình triển khai đám mây như Public, Private, Hybrid, và Community Cloud.
  • Cloud Architecture: Cấu trúc kiến trúc đám mây, phân chia trách nhiệm giữa nhà cung cấp dịch vụ đám mây (CSP) và người dùng.

2. Cloud Data Security (Bảo mật dữ liệu trên đám mây)

  • Data Lifecycle: Hiểu các giai đoạn của vòng đời dữ liệu (Data Creation, Storage, Use, Sharing, Archival, and Destruction).
  • Data Security Strategies: Áp dụng các chiến lược bảo mật dữ liệu, bao gồm mã hóa (encryption), mã hóa dữ liệu động (data masking), và tokenization.
  • Cloud Data Storage: Bảo mật cho các cơ sở lưu trữ đám mây, kiểm tra và giám sát dữ liệu được lưu trữ.

3. Cloud Platform and Infrastructure Security (Bảo mật hạ tầng và nền tảng đám mây)

  • Infrastructure Components: Hiểu các thành phần hạ tầng như máy ảo (virtual machines), containers, hypervisors, và bảo mật của chúng.
  • Security Controls: Áp dụng các biện pháp kiểm soát bảo mật như IDS/IPS, firewalls, và quản lý bản vá (patch management).
  • Business Continuity (BC) & Disaster Recovery (DR): Lập kế hoạch liên tục kinh doanh và phục hồi sau thảm họa cho môi trường đám mây.

4. Cloud Application Security (Bảo mật ứng dụng trên đám mây)

  • Secure Software Development Lifecycle (SDLC): Tích hợp bảo mật vào quy trình phát triển phần mềm cho các ứng dụng chạy trên đám mây.
  • Application Security Testing: Phương pháp kiểm thử bảo mật ứng dụng, bao gồm DAST (Dynamic Application Security Testing) và SAST (Static Application Security Testing).
  • API Security: Bảo mật giao diện lập trình ứng dụng (API), phương thức xác thực và ủy quyền.

5. Cloud Security Operations (Vận hành bảo mật đám mây)

  • Cloud Management: Quản lý và giám sát các tài nguyên đám mây, bao gồm việc xử lý sự cố bảo mật.
  • Incident Response: Quản lý sự cố bảo mật, từ việc phát hiện đến phục hồi.
  • Log & Monitoring: Theo dõi và ghi nhận các hoạt động trên đám mây để phát hiện các hành vi đáng ngờ.

6. Legal, Risk and Compliance (Pháp lý, Rủi ro và Tuân thủ)

  • Compliance: Tuân thủ các quy định và tiêu chuẩn bảo mật cho đám mây như GDPR, HIPAA, ISO/IEC 27001.
  • Risk Management: Đánh giá rủi ro liên quan đến bảo mật đám mây, bao gồm việc định lượng và giảm thiểu rủi ro.
  • Legal Issues: Hiểu các vấn đề pháp lý liên quan đến dữ liệu và quyền riêng tư trên đám mây.

Systems Security Certified Practitioner (SSCP)

SSCP là chứng chỉ bảo mật hệ thống dành cho những người có trách nhiệm quản lý và triển khai các biện pháp bảo mật hệ thống, mạng và dữ liệu. Đây là một chứng chỉ ở mức độ nền tảng, tập trung vào việc thực hành bảo mật trong hệ thống thông tin.

1. Access Controls (Kiểm soát truy cập)

  • Authentication & Authorization: Phương pháp xác thực và phân quyền cho người dùng, bao gồm các cơ chế như Multi-Factor Authentication (MFA), SSO (Single Sign-On).
  • Access Control Models: Các mô hình kiểm soát truy cập như DAC (Discretionary Access Control), MAC (Mandatory Access Control), RBAC (Role-Based Access Control).
  • Identity Management: Quản lý danh tính và quyền truy cập, bao gồm quản lý vòng đời người dùng.

2. Security Operations and Administration (Vận hành và quản lý bảo mật)

  • Security Policies: Tạo và triển khai các chính sách bảo mật cho tổ chức.
  • Asset Management: Theo dõi và bảo vệ các tài sản CNTT (phần cứng, phần mềm, dữ liệu).
  • Change Management: Quản lý sự thay đổi trong hệ thống bảo mật để đảm bảo rằng các biện pháp kiểm soát bảo mật không bị vi phạm.

3. Risk Identification, Monitoring and Analysis (Nhận diện, Giám sát và Phân tích rủi ro)

  • Risk Assessment: Đánh giá rủi ro bảo mật dựa trên xác suất xảy ra và tác động tiềm tàng.
  • Security Monitoring: Sử dụng các công cụ và kỹ thuật giám sát an ninh mạng để phát hiện các mối đe dọa.
  • Incident Response: Xử lý sự cố bảo mật, bao gồm phân tích, báo cáo và phục hồi.

4. Cryptography (Mã hóa)

  • Encryption Algorithms: Hiểu và áp dụng các thuật toán mã hóa phổ biến như AES, RSA, và các kỹ thuật mã hóa khóa đối xứng và bất đối xứng.
  • Public Key Infrastructure (PKI): Sử dụng hệ thống PKI để quản lý các khóa mã hóa và chứng chỉ số.
  • Hashing: Sử dụng các thuật toán băm (MD5, SHA-256) để bảo vệ tính toàn vẹn của dữ liệu.

5. Network and Communications Security (Bảo mật mạng và truyền thông)

  • Network Protocols: Hiểu các giao thức mạng và các biện pháp bảo mật, bao gồm TCP/IP, DNS, HTTPS, và VPN.
  • Firewalls, IDS/IPS: Sử dụng các công cụ bảo mật mạng như firewall, hệ thống phát hiện và ngăn chặn xâm nhập (IDS/IPS).
  • Secure Communication: Bảo vệ các giao tiếp trên mạng bằng các phương pháp mã hóa và các giao thức an toàn như SSL/TLS.

6. Systems and Application Security (Bảo mật hệ thống và ứng dụng)

  • System Hardening: Tăng cường bảo mật hệ thống bằng cách loại bỏ các lỗ hổng không cần thiết, như tắt các dịch vụ không cần thiết, cập nhật bản vá.
  • Vulnerability Management: Phát hiện và xử lý các lỗ hổng bảo mật trên hệ thống và ứng dụng.
  • Secure Software Development: Áp dụng các biện pháp bảo mật trong suốt quá trình phát triển phần mềm.

7. Incident Response and Recovery (Phản ứng và phục hồi sau sự cố)

  • Incident Handling: Quản lý và phản hồi kịp thời khi phát hiện sự cố an ninh.
  • Disaster Recovery Planning (DRP): Lập kế hoạch và triển khai các phương án phục hồi sau thảm họa.
  • Backup & Restore: Thực hiện và quản lý việc sao lưu và phục hồi dữ liệu.

So sánh CCSP và SSCP

  • CCSP: Chứng chỉ cao cấp hơn, tập trung vào bảo mật cho các môi trường đám mây. Được khuyến nghị cho các chuyên gia có kinh nghiệm về quản lý bảo mật cho hạ tầng đám mây và dữ liệu.
  • SSCP: Chứng chỉ nền tảng hơn, tập trung vào quản lý bảo mật hệ thống và mạng. Thích hợp cho các chuyên gia bảo mật CNTT ở mức độ thực hành và triển khai các giải pháp bảo mật hàng ngày.

8 Data Privacy & Compliance

  • GDPR (General Data Protection Regulation): Quy định bảo vệ dữ liệu của EU, cách bảo vệ quyền riêng tư và thông tin cá nhân.
  • HIPAA (Health Insurance Portability and Accountability Act): Quy định bảo mật dữ liệu y tế và cách thực hiện các biện pháp bảo vệ thông tin sức khỏe cá nhân.
  • PCI DSS (Payment Card Industry Data Security Standard): Tiêu chuẩn bảo mật dữ liệu cho ngành công nghiệp thẻ thanh toán, bao gồm mã hóa, xác thực và bảo vệ dữ liệu khách hàng.

9. Cloud Security Practices

  • Cloud Identity and Access Management (IAM): Quản lý danh tính, quyền truy cập và xác thực trên môi trường đám mây, sử dụng các công cụ như AWS IAM, Azure AD, Google Cloud IAM.
  • Zero Trust Architecture: Mô hình bảo mật không dựa vào niềm tin mặc định, kiểm tra tất cả người dùng, thiết bị và dịch vụ truy cập mạng.
  • Cloud Encryption: Mã hóa dữ liệu trên đám mây, bao gồm mã hóa tại điểm lưu trữ (encryption at rest) và trong quá trình truyền (encryption in transit).

10. Mobile Device Security

  • MDM (Mobile Device Management): Các giải pháp quản lý và bảo mật thiết bị di động trong doanh nghiệp.
  • App Security on Mobile: Bảo mật ứng dụng di động, phân tích các lỗ hổng phổ biến như mã hóa yếu, xác thực yếu, và cách thức quản lý dữ liệu nhạy cảm.

11. Advanced Cryptography

  • Quantum Cryptography: Hiểu về các thuật toán và giao thức bảo mật trong thế giới hậu lượng tử.
  • Blockchain Security: Tính bất biến của blockchain, các rủi ro về hợp đồng thông minh (smart contract) và cách bảo vệ hệ thống tài chính phi tập trung (DeFi).

12. Threat Intelligence & Advanced Attack Detection

  • Threat Hunting: Kỹ thuật chủ động tìm kiếm các mối đe dọa tiềm ẩn trong mạng, sử dụng các công cụ như Splunk, ELK Stack.
  • Advanced Persistent Threat (APT): Các nhóm hacker có tài trợ từ nhà nước hoặc tổ chức lớn, cách thức tấn công và phòng thủ trước các mối đe dọa này.
  • SIEM (Security Information and Event Management): Các công cụ và hệ thống giám sát an ninh, như Splunk, ELK Stack, IBM QRadar.

13. IoT Security

  • IoT Vulnerabilities: Các lỗ hổng bảo mật thường gặp trong hệ thống IoT (Internet of Things), như giao tiếp không được mã hóa, xác thực yếu, và quản lý cập nhật kém.
  • IoT Device Hardening: Phương pháp bảo vệ thiết bị IoT khỏi các tấn công như botnet, malware và kiểm soát truy cập.

14. DevSecOps

  • Security Automation in DevOps Pipelines: Tích hợp các công cụ kiểm thử bảo mật như SonarQube, OWASP Dependency-Check trong quá trình phát triển phần mềm.
  • Infrastructure as Code (IaC) Security: Quản lý và bảo mật hạ tầng với Terraform, Ansible, Chef và kiểm soát cấu hình bảo mật.

15. Zero-Day Exploits and Mitigations

  • Zero-Day Vulnerabilities: Lỗ hổng chưa được phát hiện hoặc chưa được vá, cách phát hiện và giảm thiểu thiệt hại từ các cuộc tấn công dựa trên lỗ hổng zero-day.
  • Exploit Development: Phân tích và phát triển các khai thác bảo mật (exploits) nhằm hiểu rõ hơn về các biện pháp phòng vệ.

16. Digital Identity & Biometric Security

  • Biometric Authentication: Các công nghệ xác thực sinh trắc học như vân tay, nhận diện khuôn mặt và mống mắt.
  • Federated Identity Management: Quản lý danh tính liên kết giữa nhiều hệ thống hoặc tổ chức, sử dụng các giao thức như SAML, OAuth, OpenID Connect.

17. Supply Chain Security

  • Software Supply Chain Attacks: Hiểu về các cuộc tấn công nhắm vào chuỗi cung ứng phần mềm, cách phòng chống bằng các phương pháp kiểm thử mã nguồn và xác thực chữ ký số.
  • Third-party Risk Management: Quản lý rủi ro từ các bên thứ ba trong chuỗi cung ứng công nghệ thông tin.

18. Physical Security

  • Data Center Security: Bảo vệ cơ sở hạ tầng vật lý, thiết bị và trung tâm dữ liệu khỏi các rủi ro vật lý như tấn công trực tiếp, hỏa hoạn, và thiên tai.
  • Access Control Systems: Hệ thống kiểm soát truy cập vật lý, như hệ thống thẻ từ, camera an ninh, và kiểm soát quyền truy cập vật lý vào thiết bị và máy chủ.
